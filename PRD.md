# 高考倒计时 (GaoKao-Countdown) 产品需求文档 (PRD)

## 1. 项目概述
本项目旨在开发一款基于 Web 技术的高考倒计时应用，帮助学生直观地感知距离高考的时间，提供励志语录和个性化设置，营造专注备考的氛围。

## 2. 技术栈
*   **核心技术**: HTML5 + CSS3 + JavaScript (ES6+)
*   **数据存储**: IndexedDB (用于存储用户设置、自定义目标等)
*   **外部依赖**: 无 (No external libraries/frameworks like jQuery, Bootstrap, Vue, React)
*   **兼容性**: Chrome 61+

## 3. 页面流程与结构
*   **页面顺序**: 加载页 (Loading Page) -> 首页 (Home Page)
*   **文件结构**:
    ```text
    /
    ├── index.html          # 入口文件
    ├── styles/             # 样式文件夹
    │   ├── main.css        # 主样式
    │   ├── background.css  # 背景样式
    │   ├── card.css        # 卡片样式 (液态玻璃效果)
    │   ├── settings.css    # 设置弹窗样式
    │   ├── themes.css      # 主题样式
    │   └── notice.css      # 提示信息样式
    └── scripts/            # 脚本文件夹
        ├── main.js         # 主逻辑
        ├── loading.js      # 加载页逻辑
        ├── time.js         # 倒计时逻辑
        ├── inspire.js      # 励志语录加载逻辑
        ├── settings.js     # 设置逻辑
        └── notice.js       # 提示信息逻辑
    ```

## 4. 核心功能逻辑

### 4.1 倒计时逻辑
根据当前时间与高考时间的对比，显示不同的倒计时状态：
1.  **考前阶段 (6月7日前)**:
    *   显示文本: "距离 xxxx 年高考还有..."
    *   目标时间: 当年 6月7日 9:00 am
2.  **考试期间 (6月7日 - 6月9日)**:
    *   显示文本: "距离 xxxx 年高考结束还有..."
    *   目标时间: 当年 6月9日 18:30 pm
3.  **考后阶段 (6月9日后)**:
    *   显示文本: "距离 xxxx 年(第二年)高考还有..."
    *   目标时间: 次年 6月7日 9:00 am

### 4.2 设置与个性化 (Preferences)
设置面板支持以下功能：
*   **主题设置**:
    *   **模式**: 白天模式 / 夜间模式 / 跟随系统
    *   **主题色**: 内置双色可选 (蓝色、橙色为基调)
*   **背景设置**:
    *   **来源**: 内置颜色 / Bing每日壁纸 （默认）
    *   **特效**: 下雪动效开关
*   **显示设置**:
    *   **倒计时精度**: 是否显示毫秒（开关）
    *   **一言句子类型**: 选择励志语录的类型（可选择的标签）
*   **动画与交互**:
    *   **开启动画**: 开关
    *   **液态玻璃效果**: 开关 (CSS + SVG 滤镜实现)
*   **目标设置**:
    *   可自定义"事件名称" (默认"高考")
    *   可自定义"目标日期" (默认逻辑如 4.1 所述)

## 5. UI/UX 设计规范
*   **主色调**: 蓝色、橙色
*   **设计风格**:
    *   **卡片**: 统一采用"液态玻璃"样式 (Liquid Glass)，使用 CSS backdrop-filter 和 SVG 滤镜实现流体/扭曲的玻璃质感。
    *   **弹窗**: 设置弹窗的显隐采用 Mac OS 的 "神奇效果" (Genie Effect)。
    *   **控件**: 开关 (Switch) 样式统一模仿 iOS 风格。
*   **交互**: 追求流畅的动画体验，注重细节反馈。

## 6. 数据存储
使用 **IndexedDB** 存储用户的偏好设置，确保刷新页面后配置不丢失：
*   用户选择的主题、背景模式
*   自定义的目标日期和事件名称
*   开关状态 (毫秒、特效等)
*   一言句子类型选择

## 7. 浏览器兼容性
*   确保在 Chrome 61+ 及主流现代浏览器上正常运行。
*   使用标准 Web API，避免过度依赖实验性特性。
